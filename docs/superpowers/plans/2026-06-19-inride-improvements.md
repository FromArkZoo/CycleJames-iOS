# In-Ride Improvements Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a Free Ride (manual/gears) mode, a whole-ride intensity control, an in-ride settings panel, a post-ride review prompt, and fix the landscape ELAPSED-tile overflow — all on the `inride-improvements` branch.

**Architecture:** A new compact settings overlay (opened from the `IntervalEditBar`, which renders in both orientations) hosts a Mode toggle and a whole-ride ±W control. Free Ride switches the trainer from ERG (`setTargetPower`) to FTMS simulation @ 0% grade (`0x11`) so the bike's own gears drive resistance; ERG enforcement is gated by a pure helper. Whole-ride intensity reuses the existing `Workout.adjustingAllIntervals(byWatts:)`. The review prompt fires on natural completion, gated by a persisted completed-ride counter. Pure logic is TDD'd in a new XCTest target; SwiftUI changes are verified via Previews + simulator; live Free Ride behaviour is verified on the bike.

**Tech Stack:** SwiftUI, CoreBluetooth (FTMS), StoreKit (`requestReview`), XcodeGen (`project.yml`), XCTest via `xcodebuild`.

## Global Constraints

- iOS deployment target **17.0**; Swift **5.9**. (`project.yml`)
- Project is generated by **XcodeGen** — after any `project.yml` edit run `xcodegen generate` from the repo root before building.
- **No new third-party dependencies.**
- App target bundle id: `com.jamesbrowne.cyclejames`.
- The existing per-interval ±5 W control (`IntervalEditBar` → `adjustCurrentInterval(byWatts:)`) must remain **unchanged in behaviour**.
- Free Ride must **degrade safely**: if sim @ 0% does not work as expected on hardware, the rider still gets "ERG off + power displayed".
- Power clamps already enforced downstream: FTMS clamps watts to [0, 2000]; `Interval.adjustedByWatts` clamps to [5%, 600%] FTP. Do not duplicate clamping.
- Default ride mode is **ERG**; whole-ride offset resets to **0** at the start of each ride.

```swift
    /// Encodes an FTMS "Set Indoor Bike Simulation Parameters" payload.
    /// grade is a percentage (0 = flat). Crr/Cw use typical road defaults.
    nonisolated static func simulationParametersData(
        grade: Double,
        windSpeedMps: Double = 0,
        crr: Double = 0.0040,
        cw: Double = 0.51
    ) -> Data {
        var data = Data([opSetSimulationParameters])
        let wind = Int16(clamping: Int((windSpeedMps / 0.001).rounded())).littleEndian
        let gr = Int16(clamping: Int((grade / 0.01).rounded())).littleEndian
        let crrByte = UInt8(max(0, min(255, Int((crr / 0.0001).rounded()))))
        let cwByte = UInt8(max(0, min(255, Int((cw / 0.01).rounded()))))
        withUnsafeBytes(of: wind) { data.append(contentsOf: $0) }
        withUnsafeBytes(of: gr) { data.append(contentsOf: $0) }
        data.append(crrByte)
        data.append(cwByte)
        return data
    }

    /// Puts the trainer into simulation mode at the given grade. Used by
    /// Free Ride at grade 0 so the bike's own gears drive resistance.
    func setSimulationGrade(_ grade: Double) {
        guard hasControl, let char = controlPointChar, let p = peripheral else { return }
        let data = Self.simulationParametersData(grade: grade)
        p.writeValue(data, for: char, type: .withResponse)
    }
```
